# Quest System Upgrade (Java II)

---

## ฟีเจอร์ที่เพิ่ม

### 1. Custom Exception (InvalidQuestException)
เพิ่มคลาส `InvalidQuestException` เพื่อใช้จัดการกรณีที่ผู้ใช้กรอก Quest ID  
ไม่อยู่ในรูปแบบที่ถูกต้อง (เช่น ไม่ตรงกับรูปแบบ D-xx-x หรือ S-xx-x)  
ทำให้สามารถแยกกรณี “รูปแบบผิด” ออกจากกรณี “ไม่พบ Quest” ได้อย่างชัดเจน  
และช่วยให้โปรแกรมไม่หยุดทำงานกะทันหันเมื่อเกิดข้อผิดพลาดจากผู้ใช้

---

### 2. ระบบไอเทม (Item Requirement System)
เพิ่มระบบ Inventory โดยใช้ `HashMap<String, Integer>`  
เพื่อเก็บไอเทมของผู้เล่น และกำหนดให้ Quest บางรายการจำเป็นต้องใช้ไอเทม  
ก่อนจึงจะสามารถเริ่มทำ Quest ได้ เช่น Quest `Boss: NullPointer`  
ต้องใช้ไอเทม `KeyCard` จำนวน 1 ชิ้น  
ระบบจะตรวจสอบจำนวนไอเทมก่อนทำ Quest และหักไอเทมออกจาก Inventory  
หากไอเทมไม่เพียงพอ ระบบจะไม่อนุญาตให้ทำ Quest นั้น

---

## วิธีทดสอบระบบ (Test Steps)

1. รันโปรแกรมและกรอกรหัสนักศึกษาเป็นตัวเลข
2. เลือกเมนู Inventory เพื่อตรวจสอบว่าได้รับ `KeyCard` เริ่มต้น
3. เลือกเมนู Do quest และกรอก Quest ID ที่ต้องใช้ไอเทม
4. ระบบจะตรวจสอบและหักไอเทมออกจาก Inventory
5. ทดลองทำ Quest เดิมซ้ำเมื่อไอเทมหมด ระบบจะแจ้งเตือนว่าไอเทมไม่เพียงพอ
6. ทดลองกรอก Quest ID ผิดรูปแบบ ระบบจะแสดงข้อความจาก Custom Exception

---

## สรุป
การเพิ่ม Custom Exception และระบบไอเทม ทำให้ Quest System
